#### 示例 6:1 (Example 6:1)
a. 如果*本是一只猫*，那么*本是一个动物*。
b. *本不是一个动物*。
c. 因此，*本不是一只猫*。

#### 示例 6:2 (Example 6:2)
a. 要么*阿琳刺伤了阿尔*，要么*阿琳是清白的*。
b. *阿琳没有刺伤阿尔*。
c. 因此，*阿琳是清白的*。

如果*这条街禁止停车*且*西比尔昨晚停在那里*，那么*西比尔有麻烦了*。然而，我知道*西比尔没有麻烦*且*这条街禁止停车*。因此，*她昨晚没有停在那里*。

## 命题 (Propositions)

根据我们将要使用的*命题*（proposition）的含义，粗略地说，命题要么是一个句子，要么如果嵌套在更复杂的句子中，在被分离出来时可以成为一个句子。[^2] 它是一组具有给定含义的、可以独立存在并在被断言时产生意义的概念（或词语）。以下是示例 6:1、6:2 和 6:3 中以斜体显示的一些简单命题的例子：

#### 示例 6:4：一些基本命题 (Example 6:4: Some Basic Propositions)
a. 本是一只猫。 (Ben is a cat.)
b. 本是一个动物。 (Ben is an animal.)
c. 本不是一个动物。 (Ben is not an animal.)
d. 本不是一只猫。 (Ben is not a cat.)
e. 阿琳刺伤了阿尔。 (Arlene stabbed Al.)
f. 阿琳是清白的。 (Arlene is innocent.)
g. 阿琳没有刺伤阿尔。 (Arlene did not stab Al.)
h. 这条街禁止停车。 (Parking is prohibited on this street.)
i. 西比尔昨晚停在那里。 (Sybil parked there last night.)
j. 西比尔有麻烦了。 (Sybil is in trouble.)
k. 西比尔没有麻烦。 (Sybil is not in trouble.)
l. 她昨晚没有停在那里。 (She did not park there last night.)

请再次查看示例 6:1、6:2 和 6:3 中的论证，看看示例 6:4 中的命题在这些论证中扮演的角色。同时注意示例 6:1、6:2 和 6:3 中*如果、那么、要么、或者、以及*（*if, then, either, or, and and*）等词语的作用。

以下是一些不属于命题的事物，因为它们在断言时不能独立存在并产生意义：

#### 示例 6:5 (Example 6:5)
a. 知更鸟 (Robins)
b. 十八岁以下的人 (People under eighteen)
c. 任何人只要未满十八岁 (anybody is under eighteen)（在“如果*任何人只要未满十八岁*，那么该人不得进入”中）

示例 6:5 中的类 *a* 和 *b* 显然在单独断言时没有意义。例如，假设有人走过来对你说：“知更鸟。”你能从中理解出什么意义？当然，作为对问题的回答，比如“那些是什么鸟？”，它是有意义的。但那时说话者隐式地断言了一个完整的命题：*那些是知更鸟*，这个命题可以独立存在并在断言时产生意义。但是，如果没有这种暗示，*知更鸟*这个词本身并不是一个断言。

[^2]: 对*命题*进行更精确、更理论化的定义对目前的学习目的没有帮助。


即便示例 6:5c（*任何人只要未满十八岁*）有主语和谓语，如果它的词义与该示例括号中给出的完整句子中的词义相同，那么有人试图断言它也是没有意义的。如果有人走过来，用示例 6:5c 中的意思对你说“任何人只要未满十八岁”，你会怎么想？实际上，示例 6:5c 中引号内的命题最适合归入类逻辑，因为它的含义是*未满十八岁的人不得进入*。

## 如果-那么推理 (If-Then Reasoning)

使用*如果-那么*命题进行的推理是最重要的一种命题推理，所以我们将从它开始。下面的论证是一个*如果-那么*推理的案例，因为它采用了一个*如果-那么*命题：

a. 如果本是一只猫，那么本是一个动物。
b. 本是一只猫。
c. 因此，本是一个动物。

第一个理由 *a* 是一个*如果-那么*命题，由两个较短的命题通过词语“如果”和“那么”在适当位置连接而成。第二个理由 *b* 是对“如果”部分的单独断言。结论 *c* 是对 *a* 中“那么”部分的断言。
第一个（复杂的）命题断言，第二个理由足以让我们得出结论。结论必然地从这两个理由中得出。
这并不是说结论是真的。本很可能不是一个动物。但是，如果理由是真的，那么他必然是一个动物。也就是说，已知“如果本是一只猫，那么本是一个动物”为真，且已知“本是一只猫”也为真，那么“本必然是一个动物”。如果这两个理由是真的，就没有办法避开这个结论。因此，示例 6:6 中的论证是演绎有效的。

### 肯定前件 (Affirming the Antecedent)
在示例 6:6 中，“如果”命题已经得到了肯定（在 *b* 中）。这使我们有权得出“那么”命题（*c*）作为结论。因为“如果”部分被称为*前件*（antecedent），且因为前件在理由 *b* 中被单独肯定，示例 6:6 中的推理形式被称为*肯定前件*（affirming the antecedent）。肯定前件是一种演绎有效的形式。

### 前件与后件 (Antecedents and Consequents)
在示例 6:6 的第一个理由（*a*）中，命题*本是一个动物*是*如果-那么*命题的“那么”部分。*如果-那么*命题的“那么”部分被称为*后件*（consequent）。在示例 6:6 的第一个理由中，*本是一个动物*是后件。
在继续查看其他推理形式之前，让我们先停下来识别一下前件和后件。前件通常排在前面，但并不总是如此。有时前件排在后面而后件排在前面（示例 6:7a），有时前件被插入到后件的各部分之间（示例 6:7b）：

**示例 6:7 (Example 6:7)**
a. 如果本是一只猫，本就是一个动物。 (Ben is an animal if Ben is a cat.)
b. 西比尔，如果这条街禁止停车，就有麻烦了。 (Sybil, if parking is prohibited on this street, is in trouble.)

示例 6:6a 和 6:7a 的意思是一样的，即使顺序颠倒了。这是因为在两种情况下，“如果”这个词都附在同一个命题（*本是一只猫*）上。整个*如果-那么*命题，无论顺序如何（只要“如果”一直跟着“本是一只猫”），都告诉我们：本是一只猫这一点足以确立本是一个动物。

请注意，“如果”（if）这个词并不是前件的一部分；相反，它是前件的标志。前件是紧跟在“如果”这个词之后的命题。后件是复杂命题中的另一个单元。“那么”（then）这个词（如果被使用的话）*不是*后件的一部分。当它出现时，它是后件的一个*标志*。在示例 6:7 中，“那么”这个词被省略了，正如后件排在前面时总是被省略一样。有时即使后件排在后面它也会被省略，如“如果本是一只猫，本是一个动物”。为了使复杂命题读起来通顺，后件（*本是一个动物*）前面并不真的需要“那么”这个词。不过，它也可以出现在那里，如示例 6:6 的理由 *a* 所示。

另一个我们通常可以忽略的区别来自于代词或类似的词项对已提及事物的替换。因此（假设本是雄性），示例 6:8 中的*如果-那么*命题就我们的目的而言，在意义上基本是相同的：

**示例 6:8 (Example 6:8)**
a. 如果本是一只猫，那么本是一个动物。 (If Ben is a cat, then Ben is an animal.)
b. 如果本是一只猫，那么他是一个动物。 (If Ben is a cat, then he is an animal.)

## 小结 (Summary)

命题在被断言时可以独立存在并产生意义。当两个命题通过“如果”和“那么”等词语恰当地连接在一起时（尽管并不总是需要“那么”），结果就是一个更复杂的*如果-那么*命题。“如果”部分是*前件*，“那么”部分是*后件*。虽然前件通常排在前面，但有时后件会排在前面，有时前件会排在后件的各部分之间。在指代已提及的事物时，代词的替换通常不会改变*如果-那么*命题的意义。当一个论证由一个*如果-那么*命题作为其中一个理由、肯定的前件作为另一个理由、后件本身作为结论组成时，这个论证的形式就是*肯定前件*，这是一种演绎有效的论证形式。

## 肯定与否定前件及后件 (Affirming and Denying the Antecedent and Consequent)

除了肯定前件外，对于一个*如果-那么*命题，人们还可以进行其他三种基本的操作。其中两个是演绎无效的，一个是演绎有效的。示例 6:9 是名为*否定前件*（denying the antecedent）的操作的插图，之所以这样称呼是因为其中一个理由否定了另一个理由的前件。

**示例 6:9 (Example 6:9)**
a. 如果本是一只猫，那么本是一个动物。
b. 本不是一只猫。
c. 因此，本不是一个动物。

理由 *b* 否定了理由 *a* 的前件，结论并不必然得出。即使理由是真的，本仍有可能是一个动物。例如，他可能是一只山羊。否定前件是一种演绎无效的形式。

另一种演绎无效的形式是*肯定后件*（affirming the consequent），之所以这样称呼是因为其中一个理由肯定了另一个理由的后件，如示例 6:10 所示：

**示例 6:10 (Example 6:10)**
a. 如果本是一只猫，那么本是一个动物。
b. 本是一个动物。
c. 因此，本是一只猫。

理由 *b* 肯定了理由 *a* 的后件。同样，结论并不必然得出。理由允许本是除了猫之外的其他某种动物，尽管他是一只猫这一点足以确保他是一个动物。肯定后件是一种演绎无效的形式。

本系列中的最后一种基本形式被称为*否定后件*（denying the consequent），因为其中一个理由否定了另一个理由的后件，如示例 6:11 所示：

**示例 6:11 (Example 6:11)**
a. 如果本是一只猫，那么本是一个动物。
b. 本不是一个动物。
c. 因此，本不是一只猫。

理由 *b* 否定了理由 *a* 的后件。这一次，结论必然得出。如果你没看出来，请考虑：假设理由是真的，如果像结论的反面所说的那样，本 *是* 一只猫。那么根据理由 *a*，本就必然是一个动物。但理由 *b* 说他不是一个动物。所以，他不可能是一只猫。否定后件是一种演绎有效的形式。

## 到目前为止的小结 (Summary So Far)

到目前为止，本章通过详细阐述命题逻辑的一些基本特征，进一步探索了 *FRISCO* 中的 *I*（推理）。命题逻辑是一种演绎逻辑，在某些方面与类逻辑相似。在命题逻辑中，基本的建筑模块是命题，它由主语和谓语组成，可以独立存在并被有意义地断言。

最重要的命题逻辑关系是蕴涵（implication），如连接前件（“如果”部分）和后件（“那么”部分）的*如果-那么*命题中所发现的那样。前件和后件本身都是命题。
- 一个*如果-那么*命题表明前件命题的真足以确立后件命题的真。它表明，为了使前件为真，后件必须为真。也就是说，除非后件为真，否则前件不可能是真的。但是，一个标准的*如果-那么*命题并*不*表明后件的真足以确立前件的真，它也*不*表明为了使后件为真，前件必须为真。认为一个*如果-那么*命题表达了这些意思是常见的演绎错误。

*肯定前件*和*否定后件*是演绎有效的操作。*肯定后件*和*否定前件*是演绎无效的操作。

### 换位 (Conversion)
交换前件和后件是一个显著的变化。也就是说，示例 6:12 中的两个陈述（*a* 和 *b*）彼此非常不同。

#### 示例 6:12 (Example 6:12)
a. 如果本是一只猫，那么本是一个动物。
b. 如果本是一个动物，那么本是一只猫。

在示例 6:12*b* 中，“如果”这个词附在（即紧排在……之前）命题*本是一个动物*上。这使得*本是一个动物*变成了前件，而不是*本是一只猫*，并彻底改变了整个*如果-那么*命题的意义。新的*如果-那么*命题（示例 6:12*b*）断言本是一个动物足以确立本是一只猫。这显然不是原始命题所断言的。原始命题断言本是一只猫足以确立本是一个动物。

前件和后件之间的区别是“如果-那么”命题逻辑中最重要的区别。人们在命题逻辑中犯下的许多错误大致可以归结为混淆了这一区别，尽管通常是在更复杂的情况下。

交换*如果-那么*命题中连接基本命题的这两种不同方式是很常见的，以至于它有一个名称：*换位*（conversion）。*换位*一个*如果-那么*命题就是交换其前件和后件。示例 6:12*b* 是示例 6:12*a* 的换位命题（converse），示例 6:12*a* 是示例 6:12*b* 的换位命题。

前件和后件相对于彼此的位置与它们是否被交换无关。唯一重要的是“如果”这个词的位置。因此，示例 6:13 中的两个命题也互为换位命题：

#### 示例 6:13 (Example 6:13)
a. 如果本是一只猫，本就是一个动物。 (Ben is an animal if Ben is a cat.)
b. 如果本是一个动物，本是一只猫。 (If Ben is an animal, Ben is a cat.)

换位是一种演绎无效的形式。它通常也是一种错误（即在“无效”的日常含义上是无效的）。

### 换质换位 (Contraposition)
一个命题的*等值换质换位命题*（contrapositive）是指交换前件和后件并将每一项都否定的命题。在示例 6:14 中，*b* 是 *a* 的等值换质换位命题。

#### 示例 6:14 (Example 6:14)
a. 如果本是一只猫，那么本是一个动物。
b. 因此，如果本不是一个动物，本就不是一只猫。

通常，等值换质换位命题在论证中可以互相替换。
从示例 6:14 中的 *a* 到 *b* 的操作被称为换质换位（contraposition）：前件和后件被交换，且每一项都被否定（negated）。更准确地说，交换要求将“如果”从一个命题转移到另一个命题，并伴随每一项的否定。结论（*b*）从理由（*a*）中得出的思维方式，与显示否定后件为演绎有效的方式相同：示例 6:14*a* 告诉我们，如果本是一只猫，那么本是一个动物。准许这一点后，让我们假设本不是一个动物。那么我们就不得不得出结论：本不是一只猫。否则，他就会是一个动物（根据 *a*）。所以，如果本不是一个动物，那么本就不是一只猫。这个*如果-那么*命题就是我们的结论（6:14*b*）。换质换位是一种演绎有效的形式。[^3]

## 否定 (Negation)
否定命题“本是一只猫”就是断言“本不是一只猫”。通常来说，人们通过引入诸如“不”（not）、“非”（non-）、“无”（no）、“不”（un-）和“并非如此”（it is not the case that）等词语或前缀来进行否定。示例 6:15 中的三个命题是对命题“本是一只猫”的否定（或否认），它们本身也是命题。就目前的学习目的而言，它们彼此基本等值：

#### 示例 6:15 (Example 6:15)
a. 本不是一只猫。 (Ben is not a cat.)
b. 并非本是一只猫。 (It is not true that Ben is a cat.)
c. 本是一个非猫。 (Ben is a non-cat.)

[^3]: 注意：对表达因果联系的*如果-那么*命题进行换质换位时要小心。要确保换质换位命题的陈述在情境中是有意义的。


示例 6:15a 是表达该思想更方便的方式。

### 双重否定 (Double Negation)
标准的双重否定规则是：*负负得正*（Two negatives make a positive）。也就是说，两个否定词互相抵消。遵循这条规则，示例 6:16 中的命题在基本意义上是等值的：

**示例 6:16 (Example 6:16)**
a. 本是一只猫。 (Ben is a cat.)
b. 说本不是一只猫是错误的。 (It is false that Ben is not a cat.)
c. 并非本不是一只猫。 (It is not true that Ben is not a cat.)
d. 并非本是一个非猫。 (It is not the case that Ben is a non-cat.)

最后三个（*b*、*c* 和 *d*）包含双重否定。双重否定互相抵消，从而得出肯定命题“本是一只猫”。思考每一个命题的意义，以确保你理解为什么最后三个命题都包含双重否定，以及为什么每一个都与第一个意思相同。

但在处理某些双重否定时可能会出现问题。例如，假设有人说“尚未证明苹果树不产果”。说这话的人可能认为这些树处于产果和不产果的中间状态，或者认为尽管它们很可能不产果，但仅仅是尚未证明而已。在这种情况下，简单地消除双重否定并得出“已经证明苹果树产果”的结论将产生误导。因此，你应该在脑海中保留一条修正规则：*负负得正，除非可能存在中间地带*。

有了这条修正规则，双重否定规则就更难使用了，但事实就是如此。语言及其使用者有时非常微妙，有时又非常直白。你必须根据实际情况做出判断。

### 双重否定与换质换位 (Double Negation and Contraposition)
有时当你对一个命题进行换质换位时，你会产生一个双重否定，而双重否定通常是可以被消除的。

**示例 6:17 (Example 6:17)**
a. 如果本是一只猫，那么本就不是一只狗。
b. 如果说本不是一只狗是错误的，那么本就不是一只猫。
c. 如果本是一只狗，那么本就不是一只猫。

在示例 6:17 中，从 *a* 到 *b* 是换质换位。从 *b* 到 *c* 是消除了 *b* 的前件中的双重否定。

消除双重否定使事情更容易理解，但必须谨慎进行，有时绝对不能这样做（正如你在上一节中所看到的）。


## 小结 (Summary)

*如果-那么*命题的换位命题（converse）是通过交换前件和后件形成的，使得前件变成后件，反之亦然。换位是一种演绎无效的操作。

*如果-那么*命题的等值换质换位命题（contrapositive）是通过交换并否定前件和后件形成的。换质换位是一种演绎有效的操作。

双重否定可以去掉，除非存在中间地带。务必小心。

## 使用字母节省时间 (Saving Time with Letters)

在命题逻辑中，我们可以通过使用字母来代表论证中的每个重要命题，从而节省组织论证的时间。一个箭头被用来显示*如果-那么*关系。这个系统不仅能帮助我们展示整体画面，还能展示各部分如何在复杂命题中组合在一起。这反过来能帮助我们判断论证是否是演绎有效的。

对于命题赋值，相当传统做法是使用小写字母，从 *p* 开始，然后是 *q*，接着是 *r*，等等。我们为每个基本命题分配不同的字母，通常使用 *p* 代表前件。以我们标准的“本”为例：
令 *p* = *本是一只猫*。
令 *q* = *本是一个动物*。

如果你愿意，你也可以分配与命题有某种联系的字母。例如，赋值可以如下所示：
令 *bc* = *本是一只猫*（*b* 代表本；*c* 代表猫）。
令 *ba* = *本是一个动物*。

通过这些赋值，我们标准的“肯定前件”示例看起来就像这样：

#### 示例 6:18 (Example 6:18)

| | | | |
| --- | --- | --- | --- |
| a. 如果本是一只猫，本就是一个动物。 | | $p \rightarrow q$ | $bc \rightarrow ba$ |
| b. 本是一只猫。 | 或 | $\underline{p}$ | $\underline{bc}$ |
| c. 因此，本是一个动物。 | 因此，$q$ | 因此，$ba$ |

*p*（或 *bc*）位于箭头的左侧这一事实表明它是 *a* 中的前件。这与 *b* 中肯定的内容相同，所以该论证是一个肯定前件的案例。你一直都知道这一点，但这个例子有助于展示如何使用字母和箭头。


此后，我将使用传统的字母赋值（*p*, *q*, *r* 等）。你可以使用任何你喜欢的字母系统。

命题 *p* 的否定由 *not p* 表示，意思是“并非 *p*”。以下是一个否定后件形式论证的符号化表示，使用了之前相同的 *p* 和 *q* 赋值：

#### 示例 6:19 (Example 6:19)
| | |
| --- | --- |
| a. 如果本是一只猫，那么本是一个动物。 | a. $p \rightarrow q$ |
| b. 本不是一个动物。 | b. $\underline{not\ q}$ |
| c. 因此，本不是一只猫。 | c. 因此，not $p$ |

有时，为包含否定词的命题分配一个字母，而不是显示一个命题的否定，会更方便。考虑示例 6:20 中的论证：

#### 示例 6:20 (Example 6:20)
a. 汤姆·杰弗斯，如果在晚上 10 点到午夜之间有人在医院候诊室看到他，就不是他做的。
b. 有人在晚上 10 点到午夜之间在医院候诊室看到了他。
c. 因此，不是他做的。

令 $p$ = *他在晚上 10 点到午夜之间在医院候诊室被看到*。
令 $q$ = *汤姆·杰弗斯没有做*。（否定词 *not* 在这里被视为 $q$ 的一部分。）

该论证可以表示为示例 6:21：

#### 示例 6:21 (Example 6:21)
a. $p \rightarrow q$
b. $\underline{p}$
c. 因此，$q$

这里有一种字母赋值，它改变了对 $q$ 的赋值，虽然对 $p$ 的赋值保持不变。

令 $p$ = *他在晚上 10 点到午夜之间在医院候诊室被看到*。
令 $q$ = *汤姆·杰弗斯做了*。（否定词在这里不被视为 $q$ 的一部分。）

在对 $q$ 的新赋值中，*not* 已从分配给 $q$ 的命题中省略。所以，在表示论证时，我们必须考虑到这一事实：

#### 示例 6:22 (Example 6:22)
a. $p \rightarrow$ not $q$
b. $\underline{p}$
c. 因此，not $q$


示例 6:21 和 6:22 是表示同一个论证的两种不同方式。我发现第一种方式更方便；你可能更喜欢第二种。只要你在整个论证分析过程中坚持你最初的赋值，两种方式都可以。要保持一致。

### 小结 (Summary)
在评估复杂的演绎论证时，为单个命题分配字母并根据这些字母来表示论证通常是有帮助的。使用箭头表示*如果-那么*关系，使用单词 *not* 表示否定。在这样做时，要确保同一个字母在整个论证中代表同一个命题。

## 合取、析取与嵌入式复杂命题 (Conjunction, Alternation, and Embedded Complex Propositions)

在下面的论证中（你在本章开头见过），两个命题被合取（conjoined）（由单词“且” *and* 连接）。这两个命题随后被视为一个整体，并成为整个蕴涵的前件。你能分配字母并判断演绎有效性吗？基本命题已用斜体标出：

**示例 6:23 (Example 6:23)**
如果*这条街禁止停车*且*西比尔昨晚停在那里*，那么*西比尔有麻烦了*。然而，我知道*西比尔没有麻烦*，且*这条街禁止停车*。因此，*她昨晚没有停在那里*。

在我下一节讨论它之前，请先尝试一下。

### 合取与嵌入式复杂命题 (Conjunction and Embedded Complex Propositions)

可以为示例 6:23 中的命题分配字母如下：
令 *p* = *这条街禁止停车*。
令 *q* = *西比尔昨晚停在那里*。
令 *r* = *西比尔有麻烦了*。

前两个命题由单词“且”连接，如 *p and q*。展示它们共同构成前件的方法是在它们周围加上括号，然后再添加显示蕴涵的箭头，如示例 6:24 所示：

**示例 6:24 (Example 6:24)**
(*p* and *q*) → *r*


因此，*p* 和 *q* 是合取的（conjoined），且该合取命题被*嵌入*（embedded）在整个蕴涵命题中。

论证的第一步是否定后件 *r*，导致对合取命题的否定：

#### 示例 6:25 (Example 6:25)
```
(p and q) → r
not r
因此，not (p and q)
```

在示例 6:25 的结论中保留括号很重要，因为被否定的是整个合取单元，而不是任何单个部分。这个结论可以读作：“并非 *p* 和 *q* 同时成立。”结论并没有告诉我们哪个部分被否定了。它只告诉我们至少有一个部分要被否定（或者两个都被否定）。论证的下一步澄清了这一点，因为原始论证断言了 *p*。下一步是从一个被否定的合取命题进行推理，其中一部分得到了肯定：

#### 示例 6:26 (Example 6:26)
```
not (p and q)（这是示例 6:25 的结论。我们在下一步中使用它。）
p
因此，not q
```

如果一个被否定的合取命题中有一部分得到了肯定，那么另一部分就必然被否定。它们不能都被接受，所以该论证是演绎有效的。示例 6:23 的结论“她昨晚没有停在那里”必然从给出的理由中得出。

这种类型的论证将再次出现在第 8 章和第 9 章关于假说反驳的内容中。它是*否定后件*和*肯定否定合取命题的一部分*的结合。最后一个标签又长又别扭，但至少它描述了所发生的事情。如果不发明另一个技术术语，就没有更好的标签了。[^4]

当一个命题的合取单独出现且被肯定时，那就暗示每个合取项（conjunct）都可以被分别肯定。例如，如果有人说“这条街禁止停车且西比尔昨天停在那里”，那么从该合取命题中可以演绎地得出：这条街禁止停车。但如果合取命题被否定，且没有其他信息，那么每一项或任一项的否定并不会演绎地得出。我们那时所知道的只是至少有一项要被否定，但我们还不知道是哪一项。

[^4]: 这是一个有争议的点，同样超出了本书的范围。如果你想深入研究，请参阅第 5 章开头提到的 Lewis, Strawson 和 Grice 的参考文献。


### 析取 (Alternation)
析取命题（alternation）由两个命题（析取项 alternants）通过单词“或者”（or）连接而成，如“默娜要回加利福尼亚，或者她很愚蠢”。对任一析取项的否定随后暗示了对另一项的断言。假设默娜不回加利福尼亚。由此得出她很愚蠢。假设她不愚蠢。由此得出她要回加利福尼亚。示例 6:27 展示了这两个演绎有效论证中的第一个。

#### 示例 6:27 (Example 6:27)
令 *p* = *默娜要回加利福尼亚*。
令 *q* = *她很愚蠢*。
| | |
| --- | --- |
| a. 默娜要回加利福尼亚，或者她很愚蠢。 | *p* or *q* |
| b. 默娜不回加利福尼亚。 | not *p* |
| c. 因此，她很愚蠢。 | 因此，*q* |

否定一个析取项；DV


示例 6:28 展示了对另一个析取项的否定：

#### 示例 6:28 (Example 6:28)
| | |
| --- | --- |
| a. 默娜要回加利福尼亚，或者她很愚蠢。 | *p* or *q* |
| b. 默娜不愚蠢。 | not *q* |
| c. 因此，默娜要回加利福尼亚。 | 因此，*p* |

否定一个析取项：DV

另一方面，对这两项析取项中任一项的肯定并不暗示对另一项的否定。例如，假设默娜 *是* 愚蠢的。那并不暗示她不回加利福尼亚。她的行为并不被该析取命题和她的愚蠢所排斥。她可能会为了某种愚蠢的理由而回去。同样地，假设她 *要* 回加利福尼亚。那与析取命题一起，并不暗示她不愚蠢。同样，她可能会为了某种愚蠢的理由而回去。示例 6:29 展示了后一种情况。

#### 示例 6:29 (Example 6:29)
| | |

| a. 默娜要回加利福尼亚，或者她很愚蠢。 | *p* or *q* |
| b. 默娜要回加利福尼亚。 | *p* |
| c. 因此，她不愚蠢。 | 因此，not *q* |

肯定弱析取项；DI

&#43; 你必须从情境中判断使用的是哪种意义的“或者”：弱意义（如示例 6:27, 6:28 和 6:29）还是强意义（如示例 6:30）。但请记住，天下没有免费的午餐。如果使用的是强意义，析取理由（*a*）就更难确立，所以如果使用的是强意义，整个论证可能会遇到麻烦。如果有疑问，我建议采用弱意义的解释。

### 小结 (Summary)
当两个命题由单词“且” *合取* 时，它们形成一个单元。如果整个单元被断言，那就暗示每个*合取项*（conjunct）都可以被分别断言。如果单元被否定（negated），那么至少有一个合取项必须被否定。在没有进一步信息的情况下，不清楚哪一个是被否定的那一个。此外，单元本身也可以是论证中的一个命题（任何复杂命题都可以）。这在符号化表示时通过在该单元周围加括号来显示。

当两个命题由单词“或者”连接时，它们是*析取项*（alternants）。如果整个析取命题被断言，那么对一个析取项的否定暗示了对另一个析取项的断言。

&#43; 然而，除非该析取命题是一个*强*析取命题，否则对一个析取项的肯定并不暗示对另一个析取项的否定。你必须从情境中确定“或者”是强还是弱。如果有疑问，我建议你选择弱意义的解释。
